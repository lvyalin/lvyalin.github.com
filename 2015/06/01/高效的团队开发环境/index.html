高效的团队开发环境

发表于 2015-06-01   |  

团队的开发环境是能够体现团队工作效率的一个重要组成部分。一个好的高效的开发环境能提升效率，减少开发过程中低级失误。

团队开发环境遇到的问题

1.无法快速搭建开发环境。尤其是项目比较庞大的情况下。体现在新同事入职，启动新项目。
2.环境的隔离差。许多人共用一个服务比如mysql，nginx等。一个同学出现低级失误，比如修改nginx错误，drop mysql库，大家都歇菜了。还有测试和开发环境的隔离。
3.线下环境和线上环境不一致。比如mysql版本，php版本，php扩展的安装等等。这个原因是线上服务一直有运维人员维护升级而开发环境没有
4.测试工程师希望能根据svn地址和版本号快速构建出一个纯洁的测试环境(不被开发染指)。且每次测试完成修复bug后，能迅速销毁重新构建，这就意味着要简单。

理想的团队开发环境

每个人有自己的多套环境（每个人处于多个分支开发中），且环境配置和线上能保持一致。能够快速简易搭建环境。

我们的解决方案

vagrant

Create and configure lightweight, reproducible, and portable development environments.
是一款用来构建虚拟开发环境的工具，非常适合 php/python/ruby/java 这类语言开发web应用，“代码在我机子上运行没有问题”这种说辞将成为历史。

如何使用Vagrant构建开发环境

构建基础box，将开发所需要的服务，比如php，mysql，redis等安装到一个机器上，打包成box。做成团队开发环境。
需要搭建新环境只需要执行三步就行
1.vagrant box add web
2.vagrant init web
3.vagrant up
一个新的环境就有了，基础box可以由运维维护。按时升级。
我们团队服务端开发，全部迁移了到vagrant，为了方便开发，弄了两台内网服务器，做了vagrant，当开发有环境需求时，直接构建并且分配虚拟机(写个shell脚本)。当一个人有在多个分支下开发的时候（小公司很常见），直接分多台虚拟机给他。这块将来可以做一个内网web服务。开发人员在一个页面上申请PHP开发环境释放PHP开发环境就OK了。离这一步其实就差个页面了。

测试的需求，其实环境有了，差的就是代码和数据了，我们本身应用框架有脚手架功能，能够一键部署代码和导入数据。所以就直接引入了。测试只需要知道svn地址和版本号，还有开发上送的数据库等数据文件就能直接构建环境测试了。测试终于不用担心在测试的过程中，研发偷偷改代码了。
